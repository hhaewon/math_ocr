1. 프로그램 소개: 개발 동기

  • 문제점: 수학 문제를 풀 때 답을 맞게 구해도 단순 계산 실수로 오답이 되는 경우가 빈번함
  • 기존의 한계: 실수를 확인하기 위해 계산기에 식을 처음부터 다시 입력해야 하는 번거로움 존재
  • 해결책: 손글씨 수식 이미지를 입력하면 자동으로 참/거짓 여부를 판정해 주는 AI 프로그램 개발

[예시: 손글씨 수식 입력 이미지]

1. 프로그램 소개: 상용 기술 및 프로젝트 목표

  • 실시간 학습 지원 기술의 발전
    • 콴다(Qanda): 문제 이미지 분석 및 풀이 검색 AI
    • Photomath: 카메라 OCR 기반 단계별 풀이 생성
    • MyScript Math: 필기 획순, 좌표, 필압 데이터를 실시간 분석
  • 본 프로젝트의 목표: 상용 기술의 핵심인 이미지 기반 수식 인식(OCR) 딥러닝 분류 모델을 직접 구현하며 인공지능의 동작 원리를 딥다이브(Deep Dive)함

2. 인공지능 기술: 지도학습 기반 이미지 분류

  • 예측·분류·군집화 중 분류(Classification) 문제에 해당
  • 왜 지도학습(Supervised Learning)인가?
    • 입력 데이터와 정답 레이블 간의 관계를 학습
    • 분류할 클래스가 명확히 정의됨 (숫자, 기호 등)
    • 비지도학습(Unsupervised)의 경우 패턴만으로 16개 기호의 미세한 차이를 정확히 구분하기 어려움

2. 인공지능 기술: 데이터셋 및 클래스 구성

  • 총 16개의 명확한 클래스 분류:
    • 숫자 (10개): $0 \sim 9$
    • 연산 기호 (4개): $+$, $-$, $\times$, $\div$
    • 기타 기호 (2개): $=$, $.$ (소수점)
  • Kaggle 손글씨 수식 데이터셋 활용
  • 클래스당 약 500장, 총 약 8,000장의 이미지 확보
  • 오인식 데이터셋 추가 보완 및 Train 80% / Val 20% / Eval (최종 평가용) 분리 학습 진행

3. 데이터 전처리 (Preprocessing)

  • AI의 특징 추출 성능을 극대화하기 위한 필수 과정
  • 자동 배경 반전: 모서리 픽셀 평균 밝기(평균 < 127)를 통해 배경색 판별
  • Otsu 이진화 (Binarization): 이미지 밝기 분포를 분석하여 흑/백을 가르는 최적의 임계값(Threshold) 자동 결정
  • 정규화 (Normalization):
    패딩 후 $32 \times 32$ 리사이징, 픽셀 값을 $0 \sim 1$ 범위로 스케일링

3. 데이터 전처리: 데이터 증강(Data Augmentation)

  • 실제 손글씨 환경의 다양한 변수(노이즈)에 대한 대응력 향상
  • 적용 기법:
    • 회전 (Rotation)
    • 이동 (Translation)
    • 확대/축소 (Zoom)
    • 전단 변환 (Shear)
  • 동일한 글씨라도 다양한 기하학적 형태를 학습하게 하여 모델의 강건성(Robustness)을 높임

4. CNN (합성곱 신경망) 설계 로직

  • 이미지의 공간적/지역적 특징 추출에 최적화된 아키텍처
  • Conv2D 층: 곡선, 모서리, 선분(예: $+$ 의 직교선, $8$ 의 폐곡선) 등 형태적 특징을 단계적(저수준→고수준)으로 학습
  • 안정화 및 과적합 방지:
    • Batch Normalization: 층 간 데이터 분포 편차를 줄여 학습 속도 및 안정성 확보
    • Dropout (0.25 / 0.5): 뉴런을 무작위로 비활성화하여 특정 뉴런에 대한 과도한 의존 방지

4. 분류 추론: Softmax 활성화 함수

  • Flatten을 통해 1차원 벡터로 변환 후, Dense 층에서 16개 클래스에 대한 점수(Logit) 도출
  • Softmax 함수를 통해 결과값을 '확률'로 변환 (총합 = $1$)
$$ \text{Softmax}(z_i) = \frac{e^{z_i}}{\sum_{j=1}^{K} e^{z_j}} $$
  • 예) 입력 "3"에 대한 모델의 최종 연산 결과:
    "3 : 97.8%, 8 : 1.3%, 5 : 0.5%, 기타 : 0.4%" ➔ "3" 채택

5. 모델의 추론: 전체 동작 흐름

  • 입력: 사용자가 작성한 1장의 수식 전체 이미지
  • Two-Stage 출력 과정:
    1. 기호 분류 (1단계): $[3, +, 8, =, 1, 1]$ 형태로 각 객체 인식
    2. 계산 검증 (2단계): "✅ 맞아요" 또는 "❌ 틀렸어요" 반환
  • CNN 모델은 각 문자 이미지를 클래스명으로 1차 분류한 뒤, 이를 실제 연산 가능한 문자열 데이터로 매핑

5. 문자 분할 (Segmentation) 알고리즘

  • Morphological Closing (형태학적 닫힘): 끊어진 획을 연결하여 객체 검출 안정화
  • findContours: 문자 영역 윤곽선 추출 후 Bounding Box 생성
  • Bounding Box 병합 (중요):
    $=$ 기호나 $\div$ 처럼 획이 분리된 경우,
    영역 간의 겹침 정도와 중심 거리 차이($\Delta x < 20$)를 계산하여 단일 기호로 병합

5. 수식 검증 (Verification) 로직

  • 분할 및 인식된 문자열(예: ['3', '+', '8', '=', '1', '1']) 조립
  • 등호(=)를 기준으로 좌변과 우변의 식 분리
  • Python 내장 eval() 함수를 통한 동적 연산 수행
  • 부동소수점 오차 제어:
    단순 == 비교가 아닌, 절댓값 차이를 이용한 검증
    if abs(left_val - right_val) < $10^{-7}$:
        print("✅ 수식이 완벽히 성립합니다!")
    else:
        print("❌ 틀린 수식입니다!")

6. 최종 성능 및 정확도 (Eval)

  • ReduceLROnPlateau(학습률 동적 감소) 및 Early Stopping(조기 종료) 기법 적용
  • 학습 과정에서 관측하지 않은 최종 Eval 데이터셋 기준 약 $98\%$ 의 높은 분류 정확도 달성
  • 새로운 사람의 손글씨에 대해서도 뛰어난 일반화(Generalization) 성능 입증

7. 시스템 한계점 분석

  • 문자 분할의 오류: 축에 평행한 직사각형(Bounding Box)의 특성상 기울어진 필체 작성 시 인접 기호의 획이 섞여 오인식 발생
  • 2차원 구조 인식 불가: 현재 1차원 문자열 배열만 지원. 분수, 지수, 루트, 적분 등 2차원 공간 배치를 가지는 복잡한 수학식 파싱의 한계
  • 정적 이미지 기반 악필 인식: 픽셀 정보만 사용하므로 글자가 심하게 겹친 경우 취약 (MyScript Math의 획순/필압 등 시계열 데이터 부재)
  • 접근 편의성: 스크린샷 저장 → 파일 첨부의 번거로운 프로세스

7. 향후 개선 아이디어

  • Rotated Bounding Box 도입: 객체의 기울기를 감지하고 회전된 경계 상자를 통해 밀집된 필기 인식률 대폭 향상
  • 수식 파싱(Parsing) 트리: 기호의 2차원 위치 정보($x, y$ 좌표) 및 연산 우선순위를 분석하여 Tree 자료구조 기반 복잡한 식 해석
  • 모바일 App 실시간 연동: 카메라 또는 태블릿 필기 화면을 실시간 인식(Real-time Inference)하여 접근 편의성 및 데이터 품질 동시 개선
